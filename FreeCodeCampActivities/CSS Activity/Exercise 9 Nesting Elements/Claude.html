    Good question! The bit about experimenting with clearing floats refers to how floats can sometimes cause unexpected layout problems if not contained properly.

    Specifically:

    • Floating elements takes them out of the normal document flow. So they can easily overlap other content, spill out of their parent container, etc.
    • If you just wanted 3 boxes floated side by side but other content below flows up around them, that's often undesirable.

    So to avoid potential layout issues like that, we need to "clear" the floats. Typically by:

    • Adding a clearfix div that has clear: both, which basically forces other content below to start below the floated boxes.

    Some things to experiment with:

    • Try removing that clearfix div and see what happens to layout with more content
    • Or change clear: both to just clear left or right and see if that still works properly
    • Or add a border around the parent container and see if floats spill out

    The goal is just to see the problems floats can introduce, and how selective clearing or clearfix techniques contain them to avoid unwanted layout behavior.
